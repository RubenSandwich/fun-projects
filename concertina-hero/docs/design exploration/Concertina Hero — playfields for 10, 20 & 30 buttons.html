Concertina Hero · playfield exploration

Room for 30 buttons

Four ways the game screen could grow past its seven toy buttons — to a 10, 20, or 30-button anglo, where half the instrument sits under each hand.

10 buttons · 1 row 20 buttons · 2 rows 30 buttons · 3 rows mockups only · nothing built

The shape of the problem

Seven lanes became thirty

Today every button owns a horizontal lane, and a lane needs roughly 40 pixels to stay readable. That works at seven. At thirty it wants 1,400 pixels of height, so the naïve scale-up is off the table — the layout has to change shape, not just stretch.

A real anglo is not a flat list of buttons. It is two hands × N rows × 5 columns, and each button already carries two notes — one on the push, one on the pull. The cards encode that already: solid for push, striped for pull. So the new axes are hand, column and row. Every mockup below is a different answer to where those three go.

Shared vocabulary

One chip per button, three ways

Three of the four concepts lean on the same encoding, so it is worth naming once. It gives thirty visually distinct buttons without thirty arbitrary colours.

Hand → colour family

Left hand cool, right hand warm. You can tell at a glance which hand a note wants before you read anything else.

Column → hue, row → shade

Five hues across, three shades deep. Light is the inner row, dark the outer — the same order your fingers reach.

Bellows → fill

Unchanged. Solid paper is a push, striped is a pull. The button says which finger, the fill says which way the bellows go.

A button is then named by position: L2b is left hand, second column, middle row. Charts would read +L2b / -R5a instead of today's +3 / -3.

Concept 01 · least change

Live Lanes

Keep everything exactly as it is, but only draw a lane for the buttons the song actually touches. A tune using nine buttons gets nine lanes, bracketed into a left-hand block and a right-hand block. The instrument can have thirty; the screen never does.

Left hand
L1b ▼G ▲A
L3a ▼C
L5b ▲D
Right hand
R1a ▼E
R3b ▲F ▼G
Reads well

Nothing about the game changes. Same scroll, same cards, same timing. The hand bracket teaches you which side to reach for.

Costs

Lane count shifts between songs, so muscle memory does too. A busy 30-button tune using twenty buttons is still twenty lanes.

Best for

Shipping something next week. It is the only concept that needs no new interaction model at all.

Concept 02 · rotate the world

Falling Columns

Turn the playfield ninety degrees. Notes fall from the top onto a row of button caps at the bottom, one thin column per button, with the two hands split by a gutter down the middle — the same gap that exists on the real instrument. Screens are wider than they are tall, so thirty columns fit where thirty lanes never could.

Shown at 20 buttons. At 30 the same field carries three columns per hue instead of two.

Reads well

Every button visible at all times, so the layout is stable across songs. Columns sit in the same left-to-right order as your fingers.

Costs

The largest rewrite: geometry, hit line, ribbon and cards all flip. Note names get cramped in a 30-pixel column, so cards lean on colour instead of text.

Best for

Players who already read Guitar Hero or Rock Band. It is the familiar idiom for many columns.

Concept 03 · look like the instrument

Keypad Reader

Stop pretending the buttons are lanes. Draw the concertina as it actually is — two keypads, each button a circle split pull-over-push, exactly like a fingering chart — and light up what to press. The upcoming beats ride a strip above, so you still see what's coming, but the screen is a picture of your hands rather than a highway.

Note names are illustrative — the real pairs come from the tuning preset. The lit key shows a push (solid); a pull would light with the striped fill.

Reads well

Scales to any button count for free, and teaches the instrument rather than the game. A chord is simply several lit circles.

Costs

Loses the approach — you feel the beat arrive rather than watch it travel, so the timing windows have to carry more weight.

Best for

A practice or learn mode. Possibly the right home for wait for correct note, which already pauses on each note.

Concept 04 · constant screen

Row-Coded Columns

Ten lanes, always: five columns for the left hand, five for the right. The row is not a lane at all — it rides on the card, as a shade and a small pip. A 10, 20 or 30-button instrument all produce exactly the same screen, so the layout never moves under you.

Left hand · columns 1–5
L2 ▼G ▲A
L4 ▼C
Right hand · columns 1–5
R1 ▲E
R3 ▼G

Pip: hollow = inner row, half = middle, filled = outer. The card's shade carries the same information at a distance, before the pip is legible.

Reads well

Ten lanes forever. Muscle memory transfers between songs and between instruments, and it fits the screen we already have.

Costs

Row becomes a second thing to decode under time pressure. Two notes in the same column on different rows collide in one lane.

Best for

Keeping the current game's feel while supporting the full instrument. The riskiest on legibility, the cheapest on screen.

A wildcard, if we want one

Bellows Split

Left-hand notes fly in from the left edge, right-hand notes from the right, and they meet at a hit line down the middle of the screen — the bellows. It is the most concertina-shaped idea on this page and the hardest to read, because your eye has to track two opposing streams. Worth a paper sketch before anyone builds it.

Before we pick one

Questions that change the answer

  • Does the seven-button toy stay the default? If yes, whatever we build has to look right at seven buttons as well as thirty, which quietly rules against Falling Columns.
  • What does a keyboard player press? Thirty buttons will not fit the number row. Two hands of keys — Q W E R T over A S D F G on the left, Y U I O P over H J K L ; on the right — mirrors the instrument and matches every layout here.
  • Do we show all buttons, or only the ones in the song? This is the real fork. Everything else follows from it.
  • Chart notation. +3 stops meaning anything on a 30-button. +L2b is unambiguous but noisier to write, and every existing chart needs a migration.
  • Mic mode is already ready for this. A 30-button anglo repeats the same pitch in several places, and aliasesOf was built for exactly that — when the mic hears a note, the game credits whichever button the chart is asking for. Duplicated notes are a layout, not an ambiguity.
